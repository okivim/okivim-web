## Requirements

### Basics
- **[Neovim](https://neovim.io/)** >= 0.11.2
- **[Git](https://git-scm.com/)** >= 2.19.0
- a **[Nerd Font](https://www.nerdfonts.com/)** to display some icons (optional)

### Required for full functionality
- **[ripgrep](https://github.com/BurntSushi/ripgrep)** and **[fd](https://github.com/sharkdp/fd)** used by `Telescope` *(more info on <a href="#dependencies">dependencies</a>)*
- **[nvim-treesitter-cli](https://github.com/tree-sitter/tree-sitter/blob/master/crates/cli/README.md)** and **C compiler** for `nvim-treesitter` *(more info on <a href="#dependencies">dependencies</a>)*
- and a **terminal that support true color and undercurl**:
  - [kitty](https://sw.kovidgoyal.net/kitty/) - (Linux / macOS)
  - [wezterm](https://wezterm.org/) - (Linux / macOS / Windows)
  - [alacritty](https://alacritty.org/) - (Linux / macOS / Windows)
  - [iterm2](https://iterm2.com/) - (macOS)
  - [ghostty](https://ghostty.org/) - (Linux / macOS / Windows)

#### Clipboard support

Neovim requires an external tool to integrate with the system clipboard.

<Tabs groupId="clipboard-os">
  <TabItem value="lin" label="Linux">
    :::info
    First, check which display server you are using:

    ```bash
    echo $XDG_SESSION_TYPE
    ```

    - If it returns `wayland` - you are using Wayland
    - If it returns `x11` - you are using X11
    :::

    <Tabs groupId="clipboard-linux">
      <TabItem value="wayland" label="Wayland (wl-clipboard)">
        ```bash
        sudo apt install wl-clipboard
        ```
      </TabItem>
      <TabItem value="x11" label="X11 (xclip)">
        ```bash
        sudo apt install xclip
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem value="mac" label="macOS">
    On macOS, Neovim uses the system clipboard through the built-in `pbcopy` and `pbpaste` utilities.

    :::info
    No additional configuration is required as long as Neovim is installed with clipboard support.
    :::

    To verify clipboard support:

    ```bash
    nvim --version | grep clipboard
    ```

    If you see `+clipboard`, Neovim is correctly configured.

    If Neovim was installed via Homebrew, clipboard support is enabled by default:

    ```bash
    brew install neovim
    ```
  </TabItem>
  <TabItem value="win" label="Windows">
    On Windows, Neovim integrates with the system clipboard using external clipboard helpers.

    <Tabs groupId="clipboard-windows">
      <TabItem value="native" label="Windows (Native)">
        Neovim uses `win32yank.exe`, which is included by default in official Neovim releases.

        :::info
        No extra setup is needed if you installed Neovim from the official installer or via winget.
        :::

        Verify clipboard support:

        ```powershell
        nvim --version | findstr clipboard
        ```
      </TabItem>

      <TabItem value="wsl" label="WSL">
        If you are using Neovim inside WSL, clipboard integration requires an additional tool.

        Install `win32yank.exe` and make sure it is available in your `$PATH`.

        Example (from inside WSL):

        ```bash
        sudo apt install wget unzip
        wget https://github.com/equalsraf/win32yank/releases/latest/download/win32yank-x64.zip
        unzip win32yank-x64.zip
        sudo mv win32yank.exe /usr/local/bin/
        ```

        If needed, add the following to your Neovim configuration:

        ```lua
        vim.g.clipboard = {
          name = "win32yank",
          copy = {
            ["+"] = "win32yank.exe -i --crlf",
            ["*"] = "win32yank.exe -i --crlf",
          },
          paste = {
            ["+"] = "win32yank.exe -o --lf",
            ["*"] = "win32yank.exe -o --lf",
          },
        }
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>
